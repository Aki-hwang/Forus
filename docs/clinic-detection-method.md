# 강남·명동·홍대 피부과 인스타 광고 수집·판별 방법

목표: 일본/중국/한국 타겟으로 **실제 피부과 클리닉이 집행 중인 인스타그램 광고**만
정확히, 충분히 많이 모은다. (인플루언서·화장품·대행사·여행 등 잡광고는 배제)

## 전체 파이프라인

```
① 검색(수집)  → ② 클리닉 판별  → ③ 중복 제거  → ④ 조회수 보강(IG)  → ⑤ 정렬/노출
  adQueries       clinics.ts        apify.ts        apify.ts(2단계)      page.tsx
```

## ① 검색 — 넓게 캐스팅 (`src/lib/adQueries.ts`)
- 소스: Meta(페이스북) 광고 라이브러리 (`curious_coder/facebook-ads-library-scraper`).
- 지역별로 **일본어(JP) + 한국어(KR)** 검색을 매주 실행. 강남은 압구정·청담·신사 포함.
- 검색어는 ISO 주차로 로테이션(주 1회 갱신). 넓게 긁고 **②에서 정밀 필터**하는 전략.

## ② 클리닉 판별 — 핵심 (`src/lib/clinics.ts`, `isExcludedAd`)
키워드 추측만으로는 인플루언서/화장품/대행사를 못 거른다. 그래서 **Meta가 페이지에
부여한 카테고리(`page_category`)를 1차 신호**로 쓴다. 3단 판별:

1. **허용 클리닉(allowlist)** — 브랜드명에 병원/클리닉 단어가 없어도 실제 클리닉인 곳
   (유앤아이·톡스앤필·블리비·쁨·리엔장 등) → **항상 유지**. 차단목록 → 제외.
2. **Meta 페이지 카테고리**
   - `Medical & health` / `Hospital` / `의료 서비스` / `Dermatologist` 등 → **확실한 클리닉, 유지**
   - `Blogger` / `Marketing agency` / `Product/service` / `Cosmetics` / `Travel` 등 → **제외**
3. **텍스트 신호** (카테고리가 없거나 `Health/beauty`처럼 애매할 때)
   - 클리닉 신호(의원/클리닉/피부과/시술/예약/皮膚科/クリニック…) → 유지
   - 잡광고 신호(크림/세럼/콜라겐/꿀팁/리뷰/마케팅/상위노출/숙소/디바이스…) → 제외
   - 둘 다 없으면 → **유지**(진짜 클리닉을 잃지 않기 위해 보수적으로)

> 왜 잘 되나: 클리닉은 거의 항상 `Medical & health`/`Hospital` 또는 이름·본문에
> 클리닉 신호를 가진다. 화장품/인플루언서/대행사는 카테고리·텍스트가 다르다.
> "Health/beauty"로 겹치는 회색지대만 텍스트로 가린다.

## ③ 중복 제거
같은 광고주가 동일 크리에이티브를 여러 ad_archive_id로 돌리므로
`광고주 + 본문 앞부분` 기준으로 중복 제거.

## ④ 조회수 보강 (2단계, `apify/instagram-scraper`)
광고 라이브러리는 조회수를 주지 않으므로, 광고주 IG 핸들의 최근 릴스
`videoPlayCount` 중앙값을 조회수로 보강 → 조회수순 정렬.

## ⑤ 노출
지역 탭(고정 트렌드) + 언어 탭(한국/일본/중국) + 정렬(조회수/팔로워/최신).
카드 클릭 → 광고주 인스타그램.

## 검증 상태 (2026-06)
- ✅ 클리닉 판별 로직: 보유 샘플 클리닉 10건 전부 유지(3건 카테고리 확정 + 7건 텍스트).
- ✅ 잡광고 텍스트 필터: 수집 109건 중 27건(화장품/인플루언서/대행/여행) 제외 → 82건.
- ⏳ **카테고리 기반 잡광고 제외의 대규모 검증**은 Apify 결제/한도 리셋(2026-07-19) 후,
  `scrapeAdDetails: true`로 카테고리까지 재수집해 라이브에서 확인 필요.

## 비용 메모
- 광고 라이브러리: $0.00075/건. 인스타 조회수: ~$2.16/1,000건.
- 주 1회 운영 시 월 ~$3~4 (무료 $5 안). 개발 중 과다 수집은 무료 한도를 빠르게 소진하므로
  로직 검증은 **수집해둔 스냅샷(`sampleAds.ts`)으로** 진행한다.
